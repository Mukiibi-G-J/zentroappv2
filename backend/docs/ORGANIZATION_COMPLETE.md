## 📖 Document Types

### By Suffix/Name Pattern

- **`README.md`** - Category overview and navigation
- **`*_SUMMARY.md`** - Implementation summaries and overviews
- **`*_GUIDE.md`** - Comprehensive how-to guides
- **`*_QUICK_START.md`** - Quick reference for getting started
- **`*_COMPLETE.md`** - Feature completion reports
- **`*_PLAN.md`** - Planning and design documents
- **`*_FIX.md`** - Bug fixes and solutions
- **`*_UPDATE.md`** - Update and change logs

3-layer access control (Role Center → Permission Sets → CRUD), JWT auth, page protection.

### 🏢 role-center/

Microsoft Business Central inspired navigation with role-based module visibility.

### 💼 sales-pilot/

Sales module implementation with invoices, posting, customers, and payments.

### 🏭 production-bom/

Production Bill of Materials for manufacturing structures and components.

### 💰 cost-structure/

Cost calculation, tracking, and allocation for items and services.

### 🔧 resources-bom/

Resource Bill of Materials for service and resource management.

### 🔄 admin-sync/

Django admin synchronization utilities and custom actions.

### 📋 general/

Miscellaneous fixes, updates, browser compatibility, and utilities.

## 📝 Maintenance

### Adding New Documentation

1. **Determine category** (reversal, permissions, general, etc.)
2. **Choose appropriate folder** in `docs/`
3. **Follow naming convention** (see Document Types above)
4. **Update category README** if it's a major addition
5. **Add to master index** (`docs/README.md`) if significant

### Creating New Categories

1. Create folder in `docs/`
2. Add `README.md` with overview
3. Add category to master `docs/README.md`
4. Move related docs into category
